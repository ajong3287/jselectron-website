# Story 1.1: Foundation Setup

**Status**: **Approved (승인)**

## Story

* **As a** 개발자,
* **I want** Next.js 14, TypeScript, Tailwind CSS 기반의 신규 프로젝트를 생성하고,
* **so that** '정직하고 안정적인' 웹사이트를 개발할 수 있는 기반을 마련한다.

## Acceptance Criteria (완료 조건)

1.  Next.js 14, TypeScript, Tailwind CSS가 설치 및 설정되어야 한다.
2.  `_shared/config-templates/`의 표준 설정 파일(ESLint, Prettier 등)이 적용되어 코드 일관성이 유지되어야 한다.
3.  기본적인 `Header` (임시 로고 사용) 및 `Footer` 컴포넌트가 생성되어야 한다.
4.  Git 저장소가 초기화되어야 한다.

## Tasks / Subtasks (개발 작업 목록)

* `[ ]` (AC 1) `npm create next-app@latest`를 사용하여 Next.js 14, TypeScript, Tailwind CSS, App Router, ESLint 프로젝트를 생성합니다.
* `[ ]` (AC 2) `_shared/config-templates/`의 Elicon 표준 `tailwind.config.ts` 및 `.eslintrc.json`을 `02_js_electronics/` 루트에 적용합니다.
* `[ ]` (AC 3) 아키텍처의 `src/components/layout/Header.tsx` 컴포넌트를 생성합니다. (임시 텍스트 로고 및 4대 메뉴 링크 포함)
* `[ ]` (AC 3) 아키텍처의 `src/components/layout/Footer.tsx` 컴포넌트를 생성합니다. (대표님이 제공해주실 기본 회사 정보 입력 영역)
* `[ ]` (AC 3) `src/app/layout.tsx` 파일을 수정하여 `Header`와 `Footer`를 전역 레이아웃에 적용합니다.
* `[ ]` (AC 4) `git init` 및 `git commit -m "Initial commit: Story 1.1 - Project foundation setup"`을 실행합니다.

## Dev Notes (개발자 참고)

* **Project Structure**: `fullstack-architecture.md`의 "7. Unified Project Structure"를 정확히 따라야 합니다.
* **Styling**: Tailwind CSS (`tailwind.config.ts`)를 사용합니다. `src/app/global.css`에 기본 Tailwind 지시문을 포함합니다.
* **Standards**: `_shared/`의 `.eslintrc.json` 코드 표준을 준수합니다.

## Testing (테스트 및 검증)

1.  `npm run dev` 실행 시 로컬 서버가 오류 없이 실행되는지 확인합니다.
2.  `npm run build` 실행 시 프로덕션 빌드가 성공하는지 확인합니다.
3.  로컬 서버 접속 시, 임시 로고가 포함된 `Header`와 `Footer`가 기본 페이지에 정상적으로 렌더링되는지 확인합니다.
