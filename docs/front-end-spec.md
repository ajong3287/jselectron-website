# UI/UX Specification: 제이에스일렉트론 (v1.0)

## 1. Overall UX Goals & Principles (전반적인 UX 목표 및 원칙)

* **Target User Personas (대상 고객)**:
    * **구매/자재 담당자**: 신속한 견적과 안정적 공급처 확인.
    * **R&D / H/W 엔지니어**: 기술/샘플 상담 및 전문성 확인.
* **Usability Goals (사용성 목표)**:
    * **신뢰도 (Trust)**: 방문 즉시 '전문가가 운영하는 안정적인 기업'임을 시각적으로 인지.
    * **명확성 (Clarity)**: "커넥터"와 "소형 배터리"라는 핵심 전문 분야가 명확히 전달.
    * **효율성 (Efficiency)**: 어떤 고객이든 3클릭 이내에 '문의하기' 도달.
* **Design Principles (디자인 원칙)**:
    * **1. 정직함 (Honesty)**: 불필요한 시각적 기교 배제, 콘텐츠 자체에 집중.
    * **2. 전문성 (Professionalism)**: 체계 잡힌 레이아웃, 일관된 타이포그래피, 안정적인 컬러 사용.
    * **3. 명확한 유도 (Clear CTA)**: '전화상담' 및 '간편 문의' 버튼을 핵심 목표로 삼아 명확하게 유도.

## 2. Information Architecture (IA, 정보 구조)

* **1. 홈 (Home)** (핵심 가치, 5대 제조사 바로가기)
* **2. 회사 소개 (About Us)** (CEO 메시지, 전문성, 파트너사)
* **3. 제품 소개 (Products)** (5대 제조사별 카테고리 페이지)
* **4. 문의하기 (Contact Us)** (전화상담 최우선, 간편 문의 양식)

## 3. User Flows (사용자 흐름)

* **Flow 1: 구매 담당자 (견적 확보 목표)**:
    * 검색 → 홈 → **회사 소개** (신뢰도 검증) → 제품 소개 → **문의하기** → **'전화상담'** (즉시 해결)
* **Flow 2: R&D 엔지니어 (기술/샘플 상담 목표)**:
    * 홈 → **제품 소개** (전문성 검증) → MUP/Amos 등 카테고리 확인 → **문의하기** → **'간편 문의'** (도면/자료 요청)

## 4. Wireframes & Mockups (와이어프레임 및 목업) - 컨셉 3.0

* **1. 홈 페이지 (전문성 + 정직함)**:
    * **상단 (Hero)**: **"커넥터/배터리 전문, 제이에스일렉트론의 1:1 맞춤 상담"** 또는 **"부품 소재 공급 달인. 무엇이든 문의하세요."** 메시지.
    * **하단 (CTA)**: **[ 전화상담 ]** / **[ 문의하기 ]** 2개 버튼 명확하게 배치.
* **2. '문의하기' 페이지 (전화 유도 중심)**:
    * **페이지 제목**: **"문의하기"**
    * **최우선 노출 (Primary CTA)**: **"전화상담"** 섹션. **"지금 바로 전화주시면 전문적인 상담이 가능합니다."** + **T. XX-XXXX-XXXX** (상담 시간) 크게 노출.
    * **차선책 (Secondary CTA)**: "전화가 어려우신가요?" 안내와 함께, **"간편 문의"** 양식을 하단에 배치. (필수: 이름, 회사, 연락처, 내용 / 선택: 이메일)

## 5. Component Library (컴포넌트 라이브러리)

* **`Button` (버튼)**: Primary(전화상담), Secondary(간편 문의) 2가지 타입.
* **`Navigation` (내비게이션)**: Header (임시 로고, 4대 메뉴), Footer (회사 기본 정보).
* **`PartnerLogoGrid` (파트너사 로고 그리드)**: 5대 제조사 로고를 **동등한 비중**으로 정렬.
* **`ProductCard` (제품 카드)**: '제품 소개' 페이지에서 제조사/카테고리 소개. (이미지 + 제목 + "상담하기" 버튼)
* **`ContactForm` (간편 문의 양식)**: '문의하기' 페이지의 최소 항목 폼 컴포넌트.

## 6. Accessibility Requirements (접근성 요구사항)

* **준수 목표**: **WCAG AA 수준**. (B2B 전문성 및 신뢰도 확보)
* **핵심 요구사항**: 명확한 색상 대비, 키보드만으로도 완벽한 조작(메뉴 이동, 폼 작성) 지원, 모든 이미지에 대체 텍스트(Alt Text) 제공.

## 7. Responsiveness Strategy (반응형 전략)

* **분기점**: 모바일, 태블릿, **데스크톱 (주력 타겟)**.
* **적응 패턴**:
    * **레이아웃**: 데스크톱(넓은 그리드), 모바일(1단 컬럼, 가독성 중심).
    * **내비게이션**: 데스크톱(가로 메뉴), 모바일(햄버거 메뉴).
    * **콘텐츠 우선순위**: 모바일에서 '전화상담' 버튼 최상단 노출.
